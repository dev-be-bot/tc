Simple Setup

Get threadcord running on your server with just one command.

Setup Command
/setup channel:#support-tickets staff_role:@Support logs_channel:#ticket-logs
• channel: Where users will send messages to create tickets
• staff_role: Which role can answer and manage tickets
• logs_channel: Where all ticket activity will be logged
1

Create Required Channels

Set up two channels: one for ticket creation and one for logs. The bot will handle everything else.

2

Define Staff Role

Select or create a role that will have permission to answer tickets (e.g., @Support, @Moderator).

3

Run /setup

Use the /setup command with your channel names and staff role. threadcord will configure everything automatically.

How Tickets Work

User Creates Ticket

A member sends any message in the ticket channel. threadcord automatically creates a private thread with ticket controls.

Staff Manages Ticket

Staff members can use the intuitive buttons to mark tickets as done, assign them, or close them.

Everything is Logged

All ticket creation, management, and closure activities are automatically logged to your designated channel.
